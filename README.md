# MyPetDiary

## サービス概要
ペットに関する日々の記録を管理するアプリです。
ペットは複数登録可能で、過去の記録を一覧表示することで
ついつい忘れがちな定期的なお世話の次の予定をチェックすることができます。

## このサービスへの思い・作りたい理由
私は犬＆猫の多頭飼いをしているのですが、それぞれお世話の内容やタイミングが異なるので、
1つのアプリに記録が集約できると便利だと思い開発することにしました。

## 想定されるユーザー層
多頭飼いをしており、ペットごとに異なるお世話（投薬、通院、お手入れ等）の管理が煩雑になっている方。

## サービスの利用イメージ
忙しい朝のルーティンで：
「ごはん」のついでに体重計に乗せた数字をサッと入力。日々の微細な変化を一覧で把握し、肥満や病気の兆候を見逃しません。
通院・お出かけの備忘録として：
「今日はワクチンを打った」「ドッグランに行った」などのイベントをその場でログ種別を選んで記録。後から「前回の予防接種はいつだっけ？」と手帳をひっくり返す手間をなくします。
多頭飼いの方の体調管理に：
「Aちゃんは薬を飲んだけど、Bちゃんはまだ」といった状況も、一覧表示することで、ケアの漏れや重複を防ぎます。

## ユーザーの獲得について
Xで紹介、今後の改善ポイントとして検索エンジン対策。

## サービスの差別化ポイント・推しポイント
既存のペット管理アプリの多くは、以下のような特徴があります：
- 1匹ごとの詳細な日記型記録が前提
- 毎日の体調や食事を細かく管理する機能が中心
- 多頭飼いで複数ペットのイベントを横断的に管理する機能は少ない
このアプリでは、多頭飼いで『イベント時のみ記録したい』というニーズに特化し、
複数ペットを登録数制限なく管理できる点を差別化としています。

* ペット複数登録を可能にして多頭飼いに対応（登録数制限なし）
* カテゴリを選んでボタンを押すだけの簡単登録
  - このアプリに向いている方・・・何かイベントがあった時だけ登録して一覧で振り返りたい方
  - このアプリに向いていない方・・・毎日体調などを細かく記録して管理したい方
* ログ記録は、日付のみor日時での登録を選べます

## 実装を予定している機能
### MVP
* ユーザー登録
* ログイン機能
* ペットの基本情報登録
* 日々のログ記録
  - お世話・お出かけなどのログを記録（種別を選択して登録）
  - 体重の記録（ログとセット、または単独で記録可能）
* 記録の閲覧機能（過去の活動と体重の変化を確認）
* ログ種別のカスタマイズ（デフォルトの種別に加えて、自分専用の種別を作成）

### その後の追加予定機能
* 体重グラフ表示
* お世話予定リマインダー機能
* ログ共有機能
